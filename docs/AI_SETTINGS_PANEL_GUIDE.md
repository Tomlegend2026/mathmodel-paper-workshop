## 🔄 工作流程示例

### 场景 1：使用 DeepSeek（推荐）

1. **点击推荐卡片**
   - 点击 "DeepSeek（推荐）" 卡片
   - 系统自动填充：
     - 提供商：DeepSeek
     - API 地址：https://api.deepseek.com/v1
     - 默认模型：deepseek-chat

2. **输入 API Key**
   - 从 DeepSeek 平台复制 API Key
   - 粘贴到 API Key 输入框

3. **选择模型**（可选）
   - 保持默认的 deepseek-chat
   - 或选择其他模型（如 deepseek-coder）

4. **测试连接**
   - 点击 "测试连接" 按钮
   - 等待几秒钟
   - 看到绿色 "连接成功" 提示

5. **保存配置**
   - 点击 "保存配置" 按钮
   - 看到 "配置保存成功" 提示
   - 完成！

---

### 场景 2：使用 Ollama（本地部署）

1. **点击推荐卡片**
   - 点击 "本地部署" 卡片
   - 系统自动填充：
     - 提供商：Ollama
     - API 地址：http://localhost:11434
     - 默认模型：qwen2.5:7b

2. **无需 API Key**
   - Ollama 不需要 API Key
   - API Key 输入框不会显示

3. **确保 Ollama 正在运行**
   - 在终端运行 `ollama serve`
   - 或确认 Ollama 后台服务已启动

4. **测试连接**
   - 点击 "测试连接" 按钮
   - 确认本地服务可访问

5. **保存配置**
   - 点击 "保存配置" 按钮
   - 完成！

---

### 场景 3：自定义配置（以通义千问为例）

1. **选择提供商**
   - 在 "选择 AI 服务商" 下拉框中选择 "通义千问"
   - 看到提供商说明："阿里云推出，国内访问稳定，支持多种场景"

2. **填写配置信息**
   - API Key：输入从阿里云获取的 API Key
   - API 地址：保持默认 `https://dashscope.aliyuncs.com/compatible-mode/v1`

3. **选择模型**
   - 在下拉框中选择合适的模型：
     - qwen-max（最强性能）
     - qwen-plus（平衡性能与成本，推荐）
     - qwen-turbo（快速响应）

4. **测试和保存**
   - 点击 "测试连接"
   - 确认成功后点击 "保存配置"

## 💡 使用技巧

### 1. 快速切换模型
- 随时可以在设置中切换不同的提供商和模型
- 无需重新输入 API Key（如果切换到同一提供商的不同模型）

### 2. 多账号管理
- 可以为不同的项目配置不同的 AI 账号
- 在浏览器中使用不同的配置文件或隐私模式

### 3. 测试连接的重要性
- 每次更换 API Key 后，建议先测试连接
- 避免在使用过程中才发现配置错误

### 4. 利用推荐配置
- 新手用户强烈建议使用推荐配置
- 减少配置错误的可能性

### 5. 查看配置指南
- 不确定如何获取 API Key？
- 查看 `docs/AI_CONFIG_GUIDE.md` 获取详细步骤

## ⚠️ 注意事项

### 安全提示
1. **不要分享 API Key**
   - API Key 相当于你的账户密码
   - 不要发送给他人或公开分享

2. **定期更换 API Key**
   - 建议每 3-6 个月更换一次
   - 如果怀疑泄露，立即更换

3. **监控用量**
   - 定期检查各平台的用量统计
   - 设置预算提醒，避免费用超支

### 常见问题

**Q: 为什么测试连接失败？**
A: 检查以下几点：
- API Key 是否正确（注意不要有多余空格）
- API 地址是否正确
- 账户是否有余额或免费额度
- 网络连接是否正常
- 是否需要科学上网（如 OpenAI）

**Q: 可以保存多个配置吗？**
A: 目前只能保存一个活跃配置，但可以随时切换。

**Q: 配置保存在哪里？**
A: 配置保存在浏览器的 localStorage 中，仅在你的设备上。

**Q: 清除浏览器缓存会影响配置吗？**
A: 会的。清除 localStorage 会删除配置，需要重新设置。
